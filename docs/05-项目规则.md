# 项目规则

## 技术栈

| 层 | 技术 |
|----|------|
| 桌面框架 | Tauri v2 |
| 前端 | React 18 + TypeScript + Vite 8 |
| UI 组件 | Ant Design 5 |
| 状态管理 | Zustand 5 |
| HTTP（浏览器回退） | Axios |
| 后端 | Rust |
| HTTP（桌面端） | reqwest |
| 数据库 | SQLite（rusqlite，bundled 编译） |

## 架构决策

1. **Zustand 而非 Redux**：API 调试工具状态简单，不需要 Redux 的模板代码
2. **Ant Design 而非 Tailwind**：更快出 UI，组件覆盖度高
3. **SQLite bundled 编译**：用户无需安装数据库，开箱即用
4. **reqwest 发 HTTP**：绕过浏览器 CORS 限制，这是 API 调试工具的核心竞争力
5. **API 不再强制属于文件夹**：folder_id 可为 NULL，根级 API 直接显示在树根

## 命名约定

- 组件文件：PascalCase（`RequestEditor.tsx`）
- 工具/服务文件：camelCase（`httpClient.ts`）
- Store 文件：`xxxStore.ts`
- Tauri 命令：snake_case（`send_http_request`）
- CSS 类名：BEM 风格前缀 `liteapi-xxx`

## 状态管理规则

- Tab 状态由 `tabStore` 管理，自动 debounce 写入 DB
- Folder/API 状态由 `folderStore` 管理，每次变更立即写 DB
- 设置状态由 `settingsStore` 管理，暂存内存
- 数据加载通过 `loadFromDb()` 一次性初始化，用 `loaded` 标志防重入
