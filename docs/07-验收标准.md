# 验收标准

## 核心功能

### 请求发送
- [ ] 输入 URL 点击 Send，能成功发送 HTTP 请求并显示响应
- [ ] 支持 GET/POST/PUT/DELETE/PATCH 五种方法
- [ ] 支持 Query 参数、Headers、Cookies 配置
- [ ] 支持 Bearer Token 和 Basic Auth 认证
- [ ] 支持 JSON/FormData/UrlEncoded/Raw/Binary 请求体
- [ ] 请求通过 Rust 后端发送，无 CORS 问题

### 响应查看
- [ ] 显示状态码、耗时、响应大小
- [ ] 显示响应头、Cookie、响应体
- [ ] 响应体支持 Pretty（JSON 美化）、Raw、Preview 模式
- [ ] 响应体可一键复制
- [ ] 固定头部不随内容滚动

### 集合管理
- [ ] 文件夹树支持无限层级
- [ ] 右键菜单：新建子文件夹、新建 API、重命名、复制、移动、删除
- [ ] 支持拖拽移动
- [ ] 搜索过滤

### 标签页
- [ ] 支持多标签页，可关闭、切换
- [ ] 新建标签页按钮在最后一个标签右侧
- [ ] 支持右键关闭、关闭其他、复制
- [ ] 修改未保存时提示
- [ ] 标签标题同步 API 名称

### 持久化
- [ ] 关闭应用再打开，数据（文件夹/API/标签页/设置）不丢失

### 国际化
- [ ] 支持中英文切换

### 主题
- [ ] 支持亮色/暗色主题，响应内容区域颜色正确

### 布局
- [ ] 侧边栏可拖拽调整宽度（180px ~ 500px）
- [ ] 请求/响应区域可拖拽调整比例（20%~85%）

## 性能
- [ ] 应用启动 < 3 秒
- [ ] 发送请求到显示响应 < 100ms（不含网络延迟）
- [ ] UI 操作（切换标签页、展开文件夹）无卡顿

## 稳定性
- [ ] 发送到不可达 URL 时有友好错误提示
- [ ] 请求超时有提示
- [ ] JSON 格式错误时有提示
- [ ] 空状态有引导提示
